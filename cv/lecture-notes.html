1. Computer Vision Fundamentals & Image Processing

AI vs Computer Vision: AI is the broad field of intelligent systems. CV is a subfield focused on enabling machines to interpret visual data. Not all AI uses vision, but modern CV heavily relies on AI.

Image Processing vs CV vs Graphics:
• Image Processing: Input = Image, Output = Enhanced/Modified Image
• Computer Vision: Input = Image, Output = Information/Understanding
• Computer Graphics: Input = Data/Models, Output = Rendered Image

Core Operations

  • Contrast Stretching: Linear mapping of intensity values to expand dynamic range
  • Thresholding & Binarization: Convert grayscale to binary (0/255) using global or adaptive thresholds
  • Histogram Equalization: Redistribute pixel intensities for better contrast

2. Convolution Operation & Feature Maps

Convolution: Sliding a kernel/filter over an image to extract spatial features (edges, textures, patterns). Mathematically: element-wise multiplication + summation.

Key Parameters:

  • Stride: Step size of kernel movement
  • Padding: Adding border pixels (Zero/Valid/Same) to control output size
  • Kernel/Filter: Learnable weights (e.g., 3×3, 5×5)

Feature Map Size Formula

Output Size = ⌊(Input - Kernel + 2×Padding) / Stride⌋ + 1

3. Neural Networks: ANN & CNN

Why ANN → CNN? Fully connected ANNs flatten images, losing spatial hierarchy. CNNs preserve spatial relationships via convolutional layers, pooling, and local receptive fields.

CNN Architecture Stack

Input Image
Conv Layers
Pooling
FC Layers

Forward Pass (Intuition)

Input → Conv + Activation → Pool → Flatten → Dense Layers → Softmax/Sigmoid → Prediction

Backpropagation (Conceptual)

1. Compute loss (prediction vs ground truth)
2. Calculate gradients ∂Loss/∂Weights using chain rule
3. Update weights: W = W - η × ∇W
4. Repeat until loss stabilizes

4. Training, Loss & Evaluation

Loss Functions

  • BCE: Binary Classification (0/1)
  • CCE: Multi-class Classification
  • MSE: Regression tasks

Optimizers

  • SGD: Simple, slow convergence
  • Adam: Adaptive learning rates, faster, default choice

Key Metrics

MetricFormulaUse Case
Accuracy(TP+TN)/TotalBalanced datasets
PrecisionTP/(TP+FP)Minimize false positives
RecallTP/(TP+FN)Minimize false negatives
F1-Score2×(P×R)/(P+R)Imbalanced data

Bias-Variance & Generalization

High Bias (Underfit)
Model too simple. Poor train & test accuracy. Fix: increase capacity, train longer.
High Variance (Overfit)
Model memorizes data. Great train, poor test accuracy. Fix: regularization, dropout, more data.
Good Generalization
Balanced bias-variance. Strong performance on unseen data.
# PyTorch Training Loop Skeleton
criterion = nn.BCEWithLogitsLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(num_epochs):
    model.train()
    for X, y in dataloader:
        optimizer.zero_grad()
        out = model(X)
        loss = criterion(out, y)
        loss.backward()
        optimizer.step()
