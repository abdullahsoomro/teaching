1 Introduction to Artificial Neural Networks (ANN)

An Artificial Neural Network is a computational model inspired by biological neurons. It consists of layers of interconnected nodes (neurons) that transform input data into predictions through weighted connections and activation functions.

Single Neuron Computation

Common Activation Functions

ReLU (Rectified Linear)

σ(z) = max(0, z)

Default choice in CNNs. Computationally cheap, avoids vanishing gradients.

Sigmoid

σ(z) = 1 / (1 + e⁻ᶻ)

Squashes to [0,1]. Used for binary output layers. Suffers from vanishing gradients.

Softmax

σ(zᵢ) = eᶻⁱ / Σeᶻʲ

Converts logits to probabilities. Used in final classification layer.

2 Why ANN → CNN for Vision?

Issue Fully Connected ANN CNN Solution
Spatial Structure Flattens 2D → 1D, destroys pixel relationships Preserves H×W×C structure throughout layers
Parameter Count 224×224×3 → 150K inputs × 4096 hidden = 614M weights Weight sharing: 16×3×3×3 = 432 weights per layer
Translation Learned features are position-specific Translation invariant: detects edges anywhere
Hierarchical Features Shallow or requires excessive depth Naturally learns: edges → textures → parts → objects

💡 Parameter Explosion Example:

A single FC layer connecting a 32×32×3 image to 100 neurons requires 32×32×3×100 = 307,200 weights. A CNN with 10 3×3 filters requires only 10×3×3×3 = 270 weights. 1,100× fewer parameters!

3 CNN Architecture Stack

A typical CNN follows a modular pipeline that progressively extracts and abstracts visual features:

Standard CNN Pipeline

Input Image → Conv2D → ReLU → MaxPool → Flatten → Dense/FC → Softmax

🔹 Convolutional Layer

Applies learnable filters to extract local features (edges, textures, patterns). Increases depth (channels).

🔸 ReLU Activation

Introduces non-linearity. Sets negative values to 0. Enables learning complex functions.

🔹 Max Pooling

Downsamples spatial dimensions. Retains strongest activations. Reduces computation & prevents overfitting.

🔸 Fully Connected

Combines extracted features for final classification. Maps high-level features to class probabilities.

4 Forward Pass: ANN vs CNN

The forward pass is the process of propagating input data through the network to generate predictions. No learning happens here—just computation.

ANN Forward Flow

h₁ = ReLU(W₁·x + b₁)

h₂ = ReLU(W₂·h₁ + b₂)

y = Softmax(W₃·h₂ + b₃)

Matrix multiplications dominate computation. All neurons connect to all inputs.

CNN Forward Flow

F₁ = Conv2D(x) → ReLU → MaxPool

F₂ = Conv2D(F₁) → ReLU → MaxPool

y = Softmax(FC(Flatten(F₂)))

Local convolutions preserve structure. Feature maps grow in depth, shrink in H×W.

5 Backpropagation: Conceptual Understanding

Backpropagation is the algorithm used to train neural networks. It calculates how much each weight contributed to the final error, then adjusts weights to reduce that error. It's essentially the chain rule applied to computation graphs.

Weight Update Rule:
wnew = wold − η · ∂L/∂w
η = learning rate, ∂L/∂w = gradient of loss w.r.t weight

Four-Step Training Loop

Step 1: Forward Pass
Input → Network → Predictions. Compute Loss (e.g., Cross-Entropy) vs Ground Truth.
Step 2: Compute Gradients
Traverse backwards using chain rule: ∂Loss/∂Output → ∂Loss/∂Weights. Frameworks do this automatically via autograd.
Step 3: Update Weights
Optimizer (SGD/Adam) applies: W = W − η·∇W. Small step in direction that reduces loss.
Step 4: Repeat
Loop over epochs. Loss decreases. Model generalizes. Stop when validation loss plateaus.

🧠 Intuition Without Calculus:

Imagine standing on a foggy mountain (loss landscape) trying to reach the valley (minimum loss). You feel the slope under your feet (gradient), take a step downhill (weight update), and repeat. The learning rate is your step size. Too big → overshoot. Too small → stuck.

6 Layer-by-Layer Shape Trace

Tracking tensor shapes is critical for debugging CNNs. PyTorch uses [Batch, Channels, Height, Width] format.

Layer Operation Input Shape Output Shape Notes
Input — — [1, 3, 32, 32] Batch=1, RGB, 32×32
Conv1 Conv2D(3→16, k=3, p=1, s=1) [1, 3, 32, 32] [1, 16, 32, 32] Padding=1 preserves H,W
ReLU1 In-place max(0,x) [1, 16, 32, 32] [1, 16, 32, 32] Shape unchanged
Pool1 MaxPool2d(k=2, s=2) [1, 16, 32, 32] [1, 16, 16, 16] H,W halved
Conv2 Conv2D(16→32, k=3, p=1, s=1) [1, 16, 16, 16] [1, 32, 16, 16] Depth doubles, H,W same
Pool2 MaxPool2d(k=2, s=2) [1, 32, 16, 16] [1, 32, 8, 8] H,W halved again
Flatten Reshape to 1D vector [1, 32, 8, 8] [1, 2048] 32×8×8 = 2048 features
FC1 Linear(2048→128) [1, 2048] [1, 128] High-level feature compression
Output Linear(128→10) + Softmax [1, 128] [1, 10] 10 class probabilities

✅ Shape Verification Tip:

Always add print(x.shape) inside forward() when debugging. Mismatched shapes are the #1 cause of PyTorch runtime errors.

7 PyTorch Implementation

1. Define CNN Architecture

import torch
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        
        # Feature extraction block
        self.conv1 = nn.Conv2d(3, 16, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(16, 32, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(kernel_size=2, stride=2)
        self.dropout = nn.Dropout(0.25)
        
        # Classification block
        self.fc1 = nn.Linear(32 * 8 * 8, 128)
        self.fc2 = nn.Linear(128, num_classes)

    def forward(self, x):
        # Input: [B, 3, 32, 32]
        x = self.pool(F.relu(self.conv1(x)))  # → [B, 16, 16, 16]
        x = self.pool(F.relu(self.conv2(x)))  # → [B, 32, 8, 8]
        x = torch.flatten(x, 1)               # → [B, 2048]
        x = F.relu(self.fc1(x))               # → [B, 128]
        x = self.dropout(x)
        x = self.fc2(x)                       # → [B, 10]
        return x

# Instantiate model
model = SimpleCNN(num_classes=10)
print(model)

8 Key Takeaways

  • ✓ ANNs connect every input to every neuron → parameter explosion for images
  • ✓ CNNs use local connectivity + weight sharing → efficient & spatially aware
  • ✓ Forward pass transforms [B,C,H,W] → [B,Classes] through Conv→Pool→FC
  • ✓ Backpropagation computes gradients automatically via autograd
  • ✓ Shape tracking is essential: always verify tensor dimensions layer-by-layer
  • ✓ ReLU + Pool + Dropout form the standard regularization & activation toolkit

📚 Summary

Architecture Concepts

  • • ANN: Global connections, high parameter count
  • • CNN: Local filters, weight sharing, translation invariant
  • • Pipeline: Conv → ReLU → Pool → Flatten → FC
  • • Depth increases, spatial dimensions decrease

Training Dynamics

  • • Forward pass: Data → Predictions
  • • Loss: Measures prediction error
  • • Backward pass: Chain rule → gradients
  • • Optimizer: Updates weights to minimize loss
