Convolution Operation & Feature Maps

Understanding the core mathematical operation behind computer vision: how convolution extracts features, the role of stride and padding, and how feature maps are generated.

Table of Contents

1 What is Convolution?

Convolution is a mathematical operation that combines two functions to produce a third function expressing how the shape of one is modified by the other. In computer vision, convolution is used to extract local features (edges, textures, patterns) from images by sliding a small matrix called a kernel or filter over the input image.

🎯 Intuition:

"Convolution is like a flashlight scanning an image—wherever the light shines, it highlights specific patterns based on the filter's design."

Why Convolution in CV?

✅ Key Advantages

  • • Local Connectivity: Each output depends only on a small region
  • • Parameter Sharing: Same kernel applied everywhere
  • • Translation Invariance: Detects features regardless of position
  • • Hierarchical Features: Simple → complex patterns through layers

🔧 What It Extracts

  • • Edges (horizontal, vertical, diagonal)
  • • Corners and keypoints
  • • Textures and patterns
  • • Blobs and regions of interest

2 Mathematical Definition of 2D Convolution

For a 2D input image I and kernel K, the discrete convolution at position (i, j) is:

2D Convolution Formula:
(I * K)[i, j] = Σm Σn I[i+m, j+n] · K[m, n]
where m, n iterate over kernel dimensions

3 Convolution Properties

🔄 Commutativity

I * K = K * I

Order doesn't matter (theoretical); in practice we fix kernel and slide over image.

🔗 Associativity

(I * K₁) * K₂ = I * (K₁ * K₂)

Multiple convolutions can be combined into a single equivalent kernel.

➕ Distributivity

I * (K₁ + K₂) = I*K₁ + I*K₂

Convolution distributes over addition—useful for filter design.

🎯 Shift Invariance

If input shifts, output shifts by same amount

Critical for detecting features regardless of position in image.

💡 Practical Implication:

These properties enable efficient implementation (e.g., FFT-based convolution) and modular filter design (combining simple kernels to create complex detectors).

Kernel Design Principles

  • • Sum = 0: Edge detectors (respond to changes, not constant regions)
  • • Sum = 1: Smoothing filters (preserve overall brightness)

  • • Separable kernels: Can be decomposed into 1D convolutions (faster computation)

5 Stride Parameter

Stride defines how many pixels the kernel moves at each step. Stride = 1 means move 1 pixel; Stride = 2 means skip every other pixel.

Effect of Stride:
Larger stride → Smaller output → Faster computation → Less spatial detail

6 Padding Strategies

Padding adds extra pixels around the input border to control output size and preserve edge information.

Type Description Output Size Use Case
Valid (No Padding) No extra pixels added Smaller than input When border information is not critical
Same (Zero Padding) Add zeros to match input size Same as input (if stride=1) Preserving spatial dimensions through layers
Reflect Mirror input at borders Configurable Reducing border artifacts in image generation
Replicate Repeat edge pixel values Configurable Maintaining edge continuity

8 Output Size Formula

The spatial dimensions of the output feature map can be calculated without performing convolution:

Output Size Formula:
O = ⌊(I - K + 2P) / S⌋ + 1
O = output dimension
I = input dimension
K = kernel size
P = padding
S = stride
⌊⌋ = floor function

Worked Examples

Example 1: Standard Conv

Input=32×32, Kernel=5, Padding=2, Stride=1

O = ⌊(32 - 5 + 2×2) / 1⌋ + 1
O = ⌊(32 - 5 + 4)⌋ + 1
O = ⌊31⌋ + 1 = 32

→ Output: 32×32 (same size)

Example 2: Downsampling

Input=28×28, Kernel=3, Padding=0, Stride=2

O = ⌊(28 - 3 + 2×0) / 2⌋ + 1
O = ⌊25 / 2⌋ + 1
O = ⌊12.5⌋ + 1 = 13

→ Output: 13×13 (≈½ resolution)

Example 3: Large Kernel

Input=100×100, Kernel=11, Padding=5, Stride=1

O = ⌊(100 - 11 + 2×5) / 1⌋ + 1
O = ⌊99⌋ + 1 = 100

→ Output: 100×100 (padding preserves size)

Example 4: Asymmetric

Input=64×48, Kernel=7×3, Padding=3×1, Stride=2×1

Oh = ⌊(64-7+6)/2⌋+1 = ⌊31.5⌋+1 = 32
Ow = ⌊(48-3+2)/1⌋+1 = ⌊47⌋+1 = 48

→ Output: 32×48 (height downsampled)

💡 Pro Tip:

Always verify output dimensions before coding! Mismatched sizes cause runtime errors in frameworks like PyTorch/TensorFlow.

9 Multi-Channel Convolution (RGB Images)

Real images have 3 channels (RGB). Convolution extends naturally: the kernel also has 3 channels, and outputs are summed across channels.

Multi-Channel Convolution:
Output[i,j] = Σc=1C Σm,n Ic[i+m,j+n] · Kc[m,n]
C = number of input channels (e.g., 3 for RGB)

Key Points

  • • Each kernel produces one feature map, regardless of input channels
  • • To get N output feature maps, use N different kernels
  • • Kernel depth must match input depth (3 for RGB, 1 for grayscale)
  • • This is why CNN layers specify: in_channels, out_channels

Core Concepts

  • • Stride controls output resolution
  • • Padding preserves spatial dimensions
  • • Output size formula: O = ⌊(I-K+2P)/S⌋+1
